Creating Exercises

This page contains examples of the different types of exercises you can create from a single code snippet. Use this as a reference as you fill out your Snippet Study Gallery over the next 10 weeks and beyond.

menu

PyTut Link

Include a PyTut link to your working snippet for easy access to step-through studying.



top

Working Snippet

Embed an example of the complete working snippet. There are many ways to pass the same tests, including a complete snippet for reference will help anyone studying your exercises make sure they're reinforcing the right concepts.
(yes, they could just copy your answer. but that's their loss)



top

Parsons Problem

Just take your working snippet and shuffle the lines around. Parsons are the simplest of all these exercises to create.



top

Code Completion

Step through your code so you're confident you know what each variable should be at each step in execution, then remove a few lines.
Be sure to assert which variable was removed and what value it should have!



top

Syntax Error

Introduce a Syntax Error into your code.
A syntax error is like a mis-spelling, there is a mistake in the source code that prevents JavaScript from successfully completing the Creation Phase. Not a single line of code will execute.



top

Runtime Error

Introduce a Runtime Error into your code.
A runtime error happens when JavaScript successfully passes the Creation Phase, but your code attempts to do something that is not possible.



top

Logic Error

Introduce a Logic Error into your code.
Everything runs with no problems, but your tests fail. Your JavaScript is correct but your logic is faulty. Either the strategy you used is flawed, or you simply made a mistake implementing it. Figuring out which is the case can be very tricky.
This can be the trickiest type of error to catch and correct.
